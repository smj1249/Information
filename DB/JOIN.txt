SQL 조인(JOIN)
1. 조인의 개념
두 개 이상의 테이블을 결합하여 관련된 데이터를 하나의 결과 집합으로 반환하는 SQL 연산

관계형 데이터베이스에서 필요한 정보를 효과적으로 조회할 수 있도록 도와줌

2. 조인의 종류
(1) 내부 조인 (INNER JOIN)
정의: 두 테이블에서 지정된 조건을 만족하는 행만 결합하여 반환

구문:

sql
복사
편집
SELECT A.컬럼명, B.컬럼명
FROM 테이블A A
INNER JOIN 테이블B B ON A.공통컬럼 = B.공통컬럼;
예제:

구매한 회원의 이름, 상품명, 가격, 수량 조회

회원별 총 구매 금액 계산

특정 상품(예: 책)을 구매한 회원 찾기

(2) 외부 조인 (OUTER JOIN)
조인 조건에 맞지 않는 행도 포함하여 반환

✅ LEFT OUTER JOIN (왼쪽 테이블 기준)

왼쪽 테이블의 모든 행을 포함하고, 일치하는 데이터가 없는 경우 NULL 반환

구문:

sql
복사
편집
SELECT A.컬럼명, B.컬럼명
FROM 테이블A A
LEFT OUTER JOIN 테이블B B ON A.공통컬럼 = B.공통컬럼;
예제:

모든 회원의 이름과 구매 내역 조회 (구매 내역이 없는 경우 NULL)

구매 이력이 없는 회원 찾기

✅ RIGHT OUTER JOIN (오른쪽 테이블 기준)

오른쪽 테이블의 모든 행을 포함하고, 일치하는 데이터가 없는 경우 NULL 반환

구문:

sql
복사
편집
SELECT A.컬럼명, B.컬럼명
FROM 테이블A A
RIGHT OUTER JOIN 테이블B B ON A.공통컬럼 = B.공통컬럼;
예제:

구매 내역을 기준으로 회원 정보 조회

회원 정보가 없는 구매 내역 찾기

✅ FULL OUTER JOIN (전체 포함)

두 테이블의 모든 행을 반환하고, 일치하는 데이터가 없는 경우 NULL 반환

구문:

sql
복사
편집
SELECT A.컬럼명, B.컬럼명
FROM 테이블A A
FULL OUTER JOIN 테이블B B ON A.공통컬럼 = B.공통컬럼;
예제:

모든 회원과 모든 구매 내역 조회 (연결되지 않은 경우 NULL)

3. 정리
INNER JOIN: 조건이 일치하는 행만 포함

LEFT JOIN: 왼쪽 테이블 모든 행 + 오른쪽 테이블에서 일치하는 행 (없으면 NULL)

RIGHT JOIN: 오른쪽 테이블 모든 행 + 왼쪽 테이블에서 일치하는 행 (없으면 NULL)

FULL JOIN: 두 테이블의 모든 행 포함 (일치하지 않는 부분은 NULL)
